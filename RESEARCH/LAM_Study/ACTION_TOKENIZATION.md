# Спецификация семантической токенизации действий

## Проблема
Жесткие селекторы (ID, XPath) делают модель хрупкой при любом изменении верстки сайта.

## Предложение: Абстрактные Токены Действий (Abstract Action Tokens)

Вместо `click('#btn-45')` модель должна генерировать токены следующего вида:

- `ACT_CLICK(SEMANTIC_LABEL='Submit', VISUAL_COORD=[x, y], CONTEXT='Payment Form')`
- `ACT_TYPE(SEMANTIC_LABEL='Email Input', VALUE='user@example.com')`
- `ACT_WAIT(CONDITION='Element_Appear', TARGET='Success Message')`

## Процесс преобразования (Mapping)
1. **LAM** генерирует семантический токен.
2. **Interface Mapper** (легкая локальная модель) сканирует текущий DOM и визуальный слой.
3. **Mapper** находит наиболее подходящий элемент по семантике и координатам.
4. **Executor** выполняет физический клик/ввод.

## Преимущество
Это позволяет LAM оперировать *смыслом* действия, а не его технической реализацией, что критически важно для обобщения (generalization) в разных средах.
